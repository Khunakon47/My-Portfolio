Personal Project

Golden Spoon Restaurant

A comprehensive restaurant management platform featuring QR-code based customer ordering, real-time kitchen communication, and staff dashboards. Includes role-based authentication (waiter, kitchen, cashier, manager) with interactive menu browsing, order tracking, and management tools. Built as a fully functional front-end prototype.

Project Overview

Golden Spoon Restaurant is a front-end prototype of a complete restaurant ordering ecosystem. Customers inside the restaurant can scan a table-specific QR code to view the menu and place orders without contacting the staff. At the same time, restaurant employees have access to a simulated management system including dashboards, order tracking, and menu control.

Although this prototype does not include backend services, all UI flows— ordering, cart management, order status updates, dashboard summaries, and menu edits—are fully functional using JavaScript-driven state simulation. The goal is to demonstrate interface design, system structure, and realistic restaurant operations.

Core Features

QR-Based Customer Ordering System

Customers access the menu through QR codes and place orders directly from their tables. Interactive menu browsing with item details, descriptions, and customization options.

Role-Based Authentication & Dashboards (4 User Types)

Waiter, Kitchen Staff, Cashier, and Manager roles with customized interfaces and permissions tailored to each position's responsibilities.

Interactive Menu, Orders & Kitchen Management

Full order management with status updates, interactive cart system, menu editing capabilities, and real-time order tracking across kitchen stations.

Technical Implementation

This project is built entirely with HTML, CSS, and JavaScript, with a focus on realistic workflows and interactive UI behavior. All data is stored and processed in the frontend using JavaScript arrays and objects.

  • Clean semantic HTML structure with reusable components
  • CSS Grid & Flexbox for scalable layout designs
  • JavaScript-powered state simulation for orders, cart, and dashboards
  • Dynamic rendering of menu items and dashboard graphs
  • Fully responsive UI for mobile-first ordering experience

Development Process & Learnings

Building both the customer and staff interfaces required designing a structure that feels real even without backend integration. A major focus was creating workflows that reflect actual restaurant operations, ensuring smooth transitions between ordering, kitchen view, and managerial tools.

Key Learning: Designing a multi-role system requires understanding how different users interact with the platform. Creating modular UI states allowed the prototype to behave like a real application despite having no server-side logic.

Future Development

  • Full backend integration with real database support
  • Real-time order updates using WebSockets
  • Authentication for customers and staff
  • Advanced analytics and sales reporting
  • Digital payments and invoice generation

Note: This version is a front-end simulation focusing on layout, UX, and user flows. Backend and database logic will be implemented in the next development phase.

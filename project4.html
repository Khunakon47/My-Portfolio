Project Overview

LibSpace is a front-end room reservation platform created to simulate how students can book meeting and conference rooms at the KKU library. The system includes fully interactive search, filtering, room browsing, booking, and booking lookupâ€” all powered through JavaScript without backend integration.

The purpose of this project is to practice front-end architecture, real-world UI structuring, and multi-step user flows while simulating realistic room availability, time slot logic, and booking validation.

Core Features

Advanced Room Search & Smart Filtering (Capacity, Amenities)

Fully functional search and filtering system using JavaScript that dynamically updates results based on room type, capacity, keywords, amenities, and availability.

4-Step Booking Wizard with Real-Time Validation

Multi-step booking process (Date Selection, Time Slot, User Details, Confirmation) with comprehensive JavaScript validation ensuring data accuracy at each step.

Time-Slot Management with Conflict Detection

Sophisticated time slot logic with unavailable dates, fully booked day handling, maximum consecutive slot limits, and disabled past time prevention for realistic booking simulation.

Email-Based Booking Lookup & Management

Users can enter their email to view bookings across three categories (Upcoming, Past, Cancelled) with full state management controlled through JavaScript.

Technical Implementation

LibSpace is built using clean semantic HTML, custom CSS, and vanilla JavaScript. All booking data, availability simulation, time slot validation, and filtering logic are implemented through in-memory objects and arrays.

  • JavaScript-driven state handling for booking flows
  • Dynamic DOM rendering for search results and room listings
  • Responsive layout using CSS Grid & Flexbox
  • Form validation for booking details
  • Modular front-end structure prepared for backend integration

Development Process & Learnings

Designing a sequential reservation workflow helped improve my ability to manage multi-step user states using JavaScript. Building realistic time-slot validation logic also strengthened my understanding of conditional rendering and UX handling for multi-state UI systems.

Key Learning: Creating an intuitive multi-step booking flow requires clear UI feedback, proper state management, and user-friendly restrictions to prevent invalid selections.

Future Development
